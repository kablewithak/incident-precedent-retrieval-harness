# Rejected Case Ideas

| Rejected idea | Rejection reason | Reason category |
|---|---|---|
| Use the supplied `connection_pool_exhaustion` source cards without controlled variants. | The acceptance audit records that the relevant source cards share the same procedure posture. The case would not establish an opposing procedure-favored candidate and therefore would miss the required adversarial property. | `non-diagnostic` |
| Create a case with tied typed evidence and different procedure availability. | A tie would test a different boundary and would not establish the required typed-selection winner that procedure metadata must fail to displace. | `non-diagnostic` |
| Derive favorable procedure availability from incident narrative language rather than card procedure fields. | Narrative interpretation would introduce an untracked free-text cue and make the oracle dependent on non-schema inference. | `contamination risk` |
| Compare candidates from different incident families to amplify the procedure contrast. | The representative-selection contract is defined for policy-compatible candidates from one family. A cross-family pool would not test procedure leakage at the intended boundary. | `non-diagnostic` |
| Add an expected representative, winner tag, or oracle rationale to the runtime input JSON. | This would break the required input/outcome separation and expose the label to runtime paths. | `contamination risk` |
| Recreate or adapt a prior calibration, E1, Tranche 01, or earlier Tranche 02 case pattern. | Prior labels, templates, and outcomes are excluded. Any reuse attempt would breach authoring independence. | `contamination risk` |
| State that a safe procedure is "better" without encoding the contrast in test-only card metadata. | Without controlled card fields and a governed mapping to source cards, the claimed contrast would lack mechanical source grounding. | `insufficient source grounding` |
| Use raw retrieval rank or score as a competing bias signal. | Retrieval rank and score are prohibited inputs and unavailable in the blind-authoring boundary. Including them would create a contaminated, non-isolated fixture. | `contamination risk` |
